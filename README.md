# Тренажёр ГОСов · 09.03.02

Локальный веб-тренажёр для подготовки к госэкзамену по направлению
**09.03.02 «Информационные системы и технологии»** (профиль РСОБ).
Парсит официальный комплект тестовых заданий из PDF и превращает его
в игру: экзамен с таймером, тренировка по дисциплинам, интервальное
повторение (SM-2) и работа над ошибками.

## Структура

```
game-gos/
├── parser/             # одноразовый Python-скрипт, PDF → JSON
│   ├── parse.py
│   ├── requirements.txt
│   └── README.md
├── data/
│   └── questions.json  # пул всех 450 вопросов (готов, в репо)
└── web/                # Vue 3 + Pinia + Vite
    ├── src/
    └── package.json
```

## Запуск

```bash
cd web
npm install
npm run dev
```

Откроется `http://localhost:5173/`. Прогресс хранится в `localStorage`
браузера — никакого бэкенда не нужно.

## Режимы

| Режим | Что делает |
|---|---|
| **Экзамен** | Случайный билет из 22 заданий (~14 одиночных + 3–4 с несколькими ответами + 4 на соответствие/последовательность), пропорционально по 5 дисциплинам. Таймер 90 минут как на реальном экзамене. Результат и разбор в конце. |
| **Тренировка** | Фильтры по дисциплине и типам заданий. Мгновенная проверка после каждого ответа, без таймера. |
| **Повторения** | Алгоритм SM-2 (как в Anki). Каждый день — те карточки, которым «пора». Интервалы 1 → 3 → 7 → 14 → 30+ дней при правильных ответах, сброс в случае ошибки. |
| **Ошибки** | Только те вопросы, где вы ошибались. После правильного ответа вопрос автоматически уходит из этого списка. |

## Содержимое базы

5 дисциплин, 450 вопросов (по 90 на дисциплину):

- Безопасность операционных систем и баз данных — 90
- Информационный менеджмент — 90
- Разработка web-приложений на языке JavaScript — 90
- Разработка серверных приложений для WEB — 90
- Реинжиниринг бизнес-процессов — 90

Типы заданий: `single` (150), `multi` (100), `matching` (75),
`sequence` (75), `open` (50).

> **Важно:** на реальном экзамене открытых заданий (`open`) не будет,
> поэтому из всех режимов практики (экзамен, тренировка, повторение,
> работа над ошибками) они исключены — это **400** заданий. Открытые
> остаются видны только в Каталоге для сверки. Что считать
> «экзаменационным» задаётся одной константой `EXAM_TYPES`
> в `web/src/stores/questions.js`.

Экзаменационный билет (`web/src/stores/exam.js`, `EXAM_TICKET`): 22 задания —
≈14 `single` + 3–4 `multi` (случайно) + 4 `matching`/`sequence`. Внутри
одного билета вопросы не повторяются; каждый запуск — новый случайный набор.

## Перепарсинг

Если нужно перегенерировать `data/questions.json` (например, изменили
PDF), отредактируйте путь в `parser/parse.py` и запустите:

```bash
cd parser
pip install -r requirements.txt
python parse.py
```

## Сборка production

```bash
cd web
npm run build
npm run preview
```

Готовая статика появится в `web/dist/`. Благодаря `base: "./"` в
`vite.config.js` и hash-роутеру (`/#/exam`) её можно открыть из любого
каталога — и в корне домена, и в подпапке.

## Деплой и доступ с телефона

Интерфейс адаптирован под мобильные (липкая прокручиваемая навигация,
крупные элементы, без авто-зума на iOS). Прогресс хранится в `localStorage`
браузера телефона. Это чистая статика — на сервере ничего не крутится,
серверный fallback не нужен (SPA на hash-роутинге, `/#/exam`).

### Вариант 1 — Timeweb App Platform (рекомендуется, авто-деплой из GitHub)

Timeweb сам собирает проект при каждом push и раздаёт статику. Сервер
(VPS) при этом не нагружается.

**Настройка приложения** (Timeweb Cloud → **App Platform** → создать
приложение → **Фронтенд** → подключить GitHub-репозиторий):

| Поле | Значение |
|---|---|
| Тип | Фронтенд / Статический сайт |
| Репозиторий | этот репозиторий, ветка `main` (или `master`) |
| Директория приложения | `web` |
| Команда сборки | `npm ci && npm run build` |
| Каталог сборки (output) | `dist` |
| Версия Node | 20 |

> Приложение лежит в `web/`, а вопросы — в соседней `data/` (vite берёт их
> по алиасу `@data` = `../data`). App Platform клонирует **весь** репозиторий,
> поэтому при сборке из `web/` папка `data/` рядом и всё резолвится.
> Если сборка вдруг не найдёт `../data` — поставь директорию приложения `/`,
> команду `npm --prefix web ci && npm --prefix web run build`, output `web/dist`.

**Привязка домена `24kurama.ru` и SSL:**

1. В настройках приложения → **Домены** → добавить `24kurama.ru` (и при
   желании `www.24kurama.ru`). Timeweb покажет, какой `A`-record или `CNAME`
   прописать.
2. **Домены и SSL** → у `24kurama.ru` указать DNS на App Platform (по подсказке
   из шага 1). Если домен уже на NS Timeweb — записи можно добавить там же.
3. Включить бесплатный **Let's Encrypt** для `24kurama.ru` (сейчас SSL
   «Выключено»). Выпуск занимает несколько минут после того, как DNS отрезолвится.
4. Готово: `https://24kurama.ru` — открывай с телефона.

После каждого `git push` в выбранную ветку Timeweb пересоберёт и обновит сайт.

### Вариант 2 — GitHub Pages (запасной, автоматически)

В репозитории есть workflow `.github/workflows/deploy.yml`: push в
`main`/`master` → **Settings → Pages → Source: GitHub Actions** → сайт на
`https://<username>.github.io/<repo>/`.

### Вариант 3 — обычный хостинг/FTP (вручную)

```bash
cd web
npm run build
```

Залить **содержимое** `web/dist/` в корень сайта по FTP. Настраивать на
сервере ничего не нужно.
